## 共通：PHP要件

### ルール
- **PHP 7.4 以上**を必須とする。PHP 7.2 / 7.3 はサポート対象外となるため、これらの環境向けコードは修正すること。
- 推奨環境は **PHP 8.3 以上**。型宣言・named arguments・enum などの PHP 8.x 機能の積極的な活用を推奨。
- `composer.json` の `require.php` は `">=7.4"` 以上に設定すること。

## テーマ開発

### theme.json のリンクスタイル変更

**ルール**
- WordPress 7.0 から、デフォルトの `theme.json` においてリンク要素への `text-decoration: underline` の指定が**削除**された。ブラウザデフォルトと重複していたためだが、テーマ独自に上書きしている場合は意図しない表示崩れが起きる可能性がある。
- 既存テーマの `theme.json` および CSS において、リンクの `text-decoration` を明示的に指定していない箇所を確認し、必要に応じて明示的に値を設定すること。

**参照元**
- https://developer.wordpress.org/news/2026/02/whats-new-for-developers-february-2026/

---

### HtmlRenderer による div ラッパー削除

**ルール**
- Gutenberg 22.4 で導入された `HtmlRenderer` コンポーネントにより、複数のブロックにおいてエディター内の余分な `<div>` ラッパーが削除された。
- テーマの CSS セレクターが `.wp-block-xxx > div` などの形式で余分なラッパーを前提にしている場合、スタイルが崩れる可能性がある。
- フロントエンドとエディターで一貫したセレクターになっているか確認・修正すること。

**参照元**
- https://developer.wordpress.org/news/2026/02/whats-new-for-developers-february-2026/

---

### 新ブロックへのスタイル対応

**ルール**
- WordPress 7.0 で以下の新ブロックが追加された。`theme.json` および CSS にてスタイルを定義すること。
  - **Icons ブロック**
  - **Breadcrumbs ブロック**
  - **Heading ブロック**（既存の Heading とは別に独立ブロック化）
- **Grid ブロック**はレスポンシブ対応が強化された。既存のグリッドレイアウト実装と競合しないか確認すること。
- **Cover ブロック**は動画背景の埋め込みに対応。関連スタイルの追加を検討すること。

**参照元**
- https://wordpress.org/news/2026/02/wordpress-7-0-beta-1/
- https://developer.wordpress.org/news/2026/03/whats-new-for-developers-march-2026/

## プラグイン開発

### AI Connectors API への対応

**ルール**
- WordPress 7.0 では `php-ai-client` パッケージを基盤とした **Connectors 機能**が Core に導入された。
- AI 機能をプラグインに組み込む場合は、独自実装ではなく `php-ai-client` の共通インターフェースを使用することを強く推奨する。これにより、OpenAI / Google / Anthropic などのプロバイダー切り替えが設定変更のみで対応できる。
- 管理画面の `設定 > Connectors` でユーザーが外部AI接続を一元管理できるようになる。プラグインはこの UI に統合できるよう、拡張ポイント（ルートベースアーキテクチャ）を活用すること。
- プロバイダープラグイン:
  - OpenAI: https://wordpress.org/plugins/openai-provider/
  - Google: https://wordpress.org/plugins/google-ai-provider/
  - Anthropic: https://wordpress.org/plugins/anthropic-ai-provider/

**参照元**
- https://developer.wordpress.org/news/2026/03/whats-new-for-developers-march-2026/
- https://wordpress.org/news/2026/02/wordpress-7-0-beta-2/

---

### DataViews への対応

**ルール**
- WordPress 7.0 では管理画面の一覧表示に **DataViews** が採用され、モダンなアプリライクなインターフェースに刷新された。
- プラグインが WP_List_Table を継承してカスタム一覧画面を実装している場合、DataViews ベースへの移行を検討すること（7.0時点では既存 WP_List_Table は引き続き動作するが、将来の廃止に備える）。
- DataViews にはサードパーティタイプを登録するための基盤が整備されつつある（将来のリリースで正式対応予定）。

**参照元**
- https://wordpress.org/news/2026/02/wordpress-7-0-beta-1/

---

### WP-CLI 新コマンドへの対応

**ルール**
- `wp block` コマンド（ブロックエンティティへの読み取りアクセス・パターン/テンプレートのエクスポート）が追加された。
- `wp ability` コマンドも開発中。
- CI/CD パイプラインや開発ツールで WP-CLI を使用している場合は、新コマンドの活用を検討すること。

**参照元**
- https://developer.wordpress.org/news/2026/03/whats-new-for-developers-march-2026/

---

### テンプレートファイルの読み込み方法

**ルール**
- `template_include` フィルターはプリミティブな文字列のみを受け付ける。"stringable objects"（`__toString()` を持つオブジェクト）を使ったテンプレートパスの指定は**公式サポート外**であり、WordPress 6.9.2 のセキュリティパッチでサイトが白画面になる不具合の原因となった。
- テンプレートのパスは必ずプリミティブ文字列で渡すこと。

**参照元**
- https://wordpress.org/news/2026/03/wordpress-6-9-3-and-7-0-beta-4/

## ブロック開発

### ブロック API バージョン

**ルール**
- 新規ブロックは **API バージョン 3 以上**を使用すること。
- API バージョン 3 未満のブロックはエディターが iframe 化されない動作となるため、将来の互換性のために更新を推奨する（※ポストエディターの常時 iframe 化は 7.1 に延期されたが、Gutenberg プラグインでは既に iframe 化される）。

**参照元**
- https://developer.wordpress.org/news/2026/02/whats-new-for-developers-february-2026/
- https://developer.wordpress.org/news/2026/03/whats-new-for-developers-march-2026/

---

### グローバル document への直接アクセス

**ルール**
- ブロックの JavaScript 内で `document.querySelector()` などグローバル `document` に直接アクセスしているコードは、エディターの iframe 化に伴い動作しなくなる可能性がある。
- エディター内での DOM 操作は `useRef` や WordPress が提供する Block Editor API を使用するよう修正すること。
- 移行ガイドを参照: https://developer.wordpress.org/block-editor/reference-guides/block-api/block-edit-save/

**参照元**
- https://developer.wordpress.org/news/2026/02/whats-new-for-developers-february-2026/

---

### Global Styles のカスタム CSS とフィーチャーセレクター

**ルール**
- Global Styles のカスタム CSS が、ブロック定義の `featureSelectors` を尊重するようになった。
- ブロックに `featureSelectors` を定義している場合、カスタム CSS の適用範囲が変わる可能性があるため動作確認を行うこと。

**参照元**
- https://developer.wordpress.org/news/2026/03/whats-new-for-developers-march-2026/

---

### ビューポートベースのブロック表示制御

**ルール**
- WordPress 7.0 ではブロックをスクリーンサイズに応じて表示/非表示にする **viewport-based visibility** が追加された。
- カスタムブロックでこの機能を利用する場合は、ブロック定義に `supports.blockVisibility` の設定を追加すること（詳細は Field Guide 公開後に確認）。

**参照元**
- https://developer.wordpress.org/news/2026/02/whats-new-for-developers-february-2026/

---

### ビジュアルリビジョン（変更追跡）

**ルール**
- エディターのリビジョンパネルに色分けされたビジュアル差分表示が追加された。
- カスタムブロックで独自の比較ロジックを持つ場合、コアの変更追跡と競合しないか確認すること。

**参照元**
- https://developer.wordpress.org/news/2026/03/whats-new-for-developers-march-2026/

## WooCommerce対応

### PHP バージョン要件の確認

**ルール**
- WordPress 7.0 の PHP 7.4+ 要件に合わせ、WooCommerce 拡張プラグインも PHP 7.4 以上を前提としたコードに移行すること。
- WooCommerce 本体のリリースノートで 7.0 対応バージョンを確認し、`requires` バージョンを更新すること。

**参照元**
- https://wordpress.org/news/2026/02/wordpress-7-0-beta-1/

---

### 管理画面 DataViews との競合確認

**ルール**
- WooCommerce は独自の管理画面（商品一覧・注文一覧など）を持つが、WordPress Core の DataViews 導入により管理画面の UI フレームワークが変化する。
- WooCommerce の管理画面拡張（カラム追加・フィルター追加など）が引き続き正常に動作するかテストすること。

**参照元**
- https://wordpress.org/news/2026/02/wordpress-7-0-beta-1/

---

### リアルタイムコラボレーションと投稿ロック

**ルール**
- WordPress 7.0 ではリアルタイムコラボレーション（RTC）が導入された。これにより従来の「投稿ロック」機能の挙動が変わる可能性がある。
- WooCommerce の商品編集画面など、投稿ロックに依存した実装がある場合は動作を確認すること。
- RTC のデータは `wp_sync_storage` という内部投稿タイプの `post_meta` に保存される。

**参照元**
- https://developer.wordpress.org/news/2026/03/whats-new-for-developers-march-2026/

## 新機能の活用指針

### リアルタイムコラボレーション（RTC）

- デフォルトでは HTTP ポーリング方式で動作。WebSocket 対応はホスト/プラグインが独自プロバイダーを追加することで実現可能。
- 同時編集者数の上限はデフォルト 2 名（`wp-config.php` の定数で変更可能）。
- 参照元: https://developer.wordpress.org/news/2026/03/whats-new-for-developers-march-2026/

### クライアントサイドメディア処理

- ブラウザ側で画像リサイズ・圧縮を処理するため、サーバー側の画像処理ロジックへの影響を確認すること。
- 参照元: https://wordpress.org/news/2026/02/wordpress-7-0-beta-1/
